# دليل اختبارات REST Client

يحتوي هذا المجلد على مجموعة من ملفات الاختبار لنقاط النهاية (Endpoints) الخاصة بالـ API باستخدام ملحق **REST Client** في VS Code.

## المتطلبات
- تثبيت ملحق [REST Client](https://marketplace.visualstudio.com/items?itemName=humao.rest-client) في VS Code.
- تشغيل السيرفر محلياً على منفذ `5000`.

## تنظيم الملفات
تم تنظيم الاختبارات في ملفات منفصلة حسب الوظيفة:
- `auth.rest`: اختبارات تسجيل الدخول وإنشاء الحسابات (Admin/User).
- `categories.rest`: اختبارات CRUD للأقسام.
- `products.rest`: اختبارات CRUD للمنتجات.
- `reviews.rest`: اختبارات نظام الآراء (رفع الصور).
- `stats.rest`: اختبارات إحصائيات لوحة التحكم.

## كيفية التشغيل
1. افتح ملف `auth.rest` أولاً وقم بتشغيل طلبات التسجيل (Register) والولوج (Login).
2. سيقوم الملحق تلقائياً بحفظ الـ **token** في متغيرات لاستخدامها في بقية الملفات.
3. يمكنك الآن فتح بقية الملفات وتشغيل الطلبات بالترتيب.

## حالات الاختبار المغطاة
- **Positive Cases (200, 201)**: التأكد من نجاح العمليات عند إرسال بيانات صحيحة.
- **Negative Cases (400, 401, 403, 404)**:
    - بيانات غير صالحة (Validation Errors).
    - الدخول بدون Token (Unauthorized).
    - محاولة مستخدم عادي الوصول لمسارات المسؤول (Forbidden).
    - محاولة الوصول لموارد غير موجودة (Not Found).

## ملاحظة حول الصور
تستخدم ملفات الـ `.rest` ميزة رفع الملفات. يرجى التأكد من وجود ملفات صور افتراضية بنفس الأسماء المذكورة في الملفات (مثل `facecream.jpg`) في نفس المجلد لضمان نجاح الاختبار.
